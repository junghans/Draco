Methods Development Environment

This page provides an overview of the methods development environment associated with the Draco CPS. Draco developers generally consider themselves to be "computational physicists" with a detailed knowledge of the physics of radiation transport, as well as knowledge of numerical methods and modern software engineering techniques. The list below presents some of the major features of the software engineering environment that Draco developers work in on a daily basis. This list is not exhaustive, but is intended to give a general idea of some of the concepts in use in the Draco project. Many of these concepts are discussed more fully elsewhere on this site.
  • Radiation Transport Methods - One, two and three dimensional numerical solutions of the Boltzmann Transport Equation
    • Particles
      • Neutral Particles - Neutrons and high energy Gamma rays
      • Charged Particles - electrons, positrons
      • Radiative Transfer - X-Ray and infra-red photons
    • Deterministic Solution Techniques
      • First and second order forms of the transport equation
      • Structured, unstructured and AMR grids
      • Spatial Differencing Techniques
        • Finite element
        • Finite difference
        • Nodal
        • Exponential
        • Discontinuous
      • Angular differencing techniques
        • Spherical Harmonics (Pn)
        • Discrete Ordinates (Sn)
        • Simplified Spherical Harmonics (SPn)
        • Fokker-Planck approximations for highly forward peaked scattering
      • Energy differencing techniques
        • Multi-group
        • Linear discontinuous for charged particle continuous-slowing-down term
      • Time differencing techniques
        • Forward Euler
        • Crank-Nicholson
        • Linear Discontinuous
      • Convergence acceleration techniques
        • MFG acceleration of fission and emission sources
        • DSA acceleration of Sn scattering sources
        • Angular multi-grid acceleration of highly forward peaked scattering
        • Discrete and continuous Fourier Analysis convergence properties
      • Automatic biasing of Monte-Carlo solutions
      • Asymptotic analysis of spatial and angular differencing methods
    • Stochastic Solution Techniques
      • Implicit Monte-Carlo
      • Linear Monte-Carlo
      • Hybrid Deterministic-Monte-Carlo Methods
  • Linear Algebra Concepts
    • Multi-grid methods
    • Conjugate-gradient and other Krylov solvers
    • Direct solvers for small, dense systems
    • Physics based preconditioners for iterative solvers
  • CASE Tools and Utilities
    • LaTeX, BibTeX, xfig - document preparation
    • CVS - version control system
    • gmake, gm4, autoconf - build system
    • DejaGnu - automatic test driver
    • Gnats - bug tracking system
    • Emacs, XEmacs - text editor and code development environment
    • Purify - a software error detection utility
    • Rational Rose - OO Design and Analysis tool
  • Programming Languages
    • Elisp - language used to define emacs templates and functions
    • C++ - object-oriented programming language
    • STL - the C++ Standard Template Library
    • Python - a scripting language
    • html - language for WWW publications
    • UML - The Unified Modeling Language for OO Design and Analysis
  • Operating Systems and Related Utilities
    • Unix, Linux - modern operating systems
    • sh, bash - the Bourne and Bourne Again shells and associated shell scripts
    • MPI - Message Passing Interface for parallel applications
    • Posix threads - light-weight processes for parallel applications
    • OpenMP - compiler directed threads implementation
  • Software Engineering Practices and Standards
    • Software Engineering Process
      • Software Quality Assurance
      • Software Life Cycle
    • Object-Oriented Analysis and Design - software design methodology consisting of abstraction, encapsulation, modularity, hierarchy, typing, concurrency, and persistence.
    • Generic Programming - use of C++ templates for static (compile-time) polymorphism
    • Levelized Design - a design composed of discrete, individually testable components
    • Design by Contract - rigorous specification of pre and post conditions at all software procedure interfaces
    • Technical Review - peer review of all work products
    • Capability Maturity Model ( CMM ) - provides a standard to measure the quality of an organization's software engineering process.
